# 최종 보고서

## 1. 프로그램 주제와 목적

내 프로그램의 주제: 다기준 의사결정(MCDA) 기반 학과 선택기

이 프로그램을 만든 이유: 자유전공학부 학생들은 대학 진학 후 수많은 전공 중 자신에게 맞는 학과를 스스로 탐색하고 설계해야 하는 주도적인 과제를 마주하게 된다. 하지만 기준이 모호하고 막연할 때가 많아 진로 선택에 어려움을 겪는다. 사용자가 주관적으로 생각하는 가치(적성, 발전가능성, 흥미)에 가중치를 부여하면, 이를 미리 저장된 학과별 고유 특성 데이터와 결합하여 정량적 점수로 산출해 주는 프로그램을 기획했다. 이를 통해 자유전공학부생들이 보다 합리적이고 객관적인 진로 의사결정을 내릴 수 있도록 돕고자 제작하였다.


---

## 2. 문제 분해 및 모델링


### 입력
사용자가 입력하는 3가지 진로 기준 중요도(가중치): 적성, 발전가능성, 흥미 (각 1점~5점의 정수)

프로그램 내부에 사전 선언된 학과 특성 데이터 (2차원 리스트)

### 처리
int() 내장 함수를 사용해 키보드로 입력받은 가중치 문자열을 정수형 데이터로 변환

반복문(for)을 사용해 2차원 리스트에 저장된 학과별 고유 점수를 행 단위로 순회

다기준 의사결정(MCDA) 가중합 알고리즘인 (학과별 점수 * 사용자의 가중치 입력값)을 각 항목마다 연산한 후 모두 합산하여 학과별 종합 점수 산출


### 출력
계산된 모든 학과의 종합 점수 결과 출력

if-elif-else 조건문을 통해 최종 점수대별로 추천 등급 표시 ([★ 강력 추천!], [우수 추천])

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름:department_data

한 행의 의미: 하나의 특정 학과가 가진 이름과 세 가지 세부 진로 특성(적성, 발전가능성, 흥미)의 고정된 고유 평가 점수 묶음

각 열의 의미:
0번 열,학과의 명칭 (문자열 자료형),"""컴퓨터공학과"""
1번 열,해당 학과가 요구하는 '적성' 부합 점수 (정수 자료형),9
2번 열,해당 학과의 전망 및 '발전가능성' 점수 (정수 자료형),10
3번 열,학생들이 느끼는 '흥미' 유발 점수 (정수 자료형),6

이 2차원 리스트를 사용한 이유: 여러 학과의 정보와 각 학과가 가진 복수의 세부 특성 점수(적성, 발전성, 흥미)를 표(Table) 형태의 구조화된 데이터로 관리하기에 가장 적합한 자료구조이기 때문이다. 하나의 리스트 안에 여러 개의 서브 리스트를 배치함으로써, 반복문을 통해 모든 학과의 다차원 데이터를 효율적으로 순회하고 일괄적으로 가중치 계산 알고리즘을 적용할 수 있었다.

## 4. 함수 설명
함수 이름,역할,입력값,반환값 또는 출력
show_intro(),"프로그램의 제목, 목적 및 가중치 입력 규칙 안내",없음,없음 (화면 텍스트 출력)
get_user_input(),사용자로부터 3가지 기준의 중요도 점수를 입력받음,없음,"사용자가 입력한 가중치 정수 리스트 [적성, 발전성, 흥미]"
find_recommendations(),2차원 리스트를 순회하며 가중합 알고리즘으로 학과별 총점 계산,"학과 데이터 리스트, 사용자 가중치 리스트","각 학과의 이름과 총점이 누적된 리스트 [[학과명, 총점], ...]"
print_result(),계산된 결과 리스트를 확인하여 조건별 추천 등급과 함께 출력,학과별 총점 결과 리스트,없음 (화면 최종 분석 텍스트 출력)

가장 중요한 함수 하나를 골라 자세히 설명하세요. 

함수 이름: find_recommendations(data, weights)

설명: 이 프로그램의 핵심 처리 단계인 다기준 의사결정(MCDA) 알고리즘을 담당하는 함수이다. 고정된 학과 데이터인 data(2차원 리스트)와 사용자가 입력한 선호도 가중치인 weights(리스트)를 매개변수로 받는다. 함수 내부에서 for row in data: 반복문을 통해 2차원 리스트의 각 행을 순회하며 학과명과 항목별 고유 점수를 변수에 할당한다. 이후 (학과점수 * 입력 가중치)를 3개 항목에 대해 각각 곱하고 합산하여 최종 종합 점수(total_score)를 구한 뒤, 결과 리스트에 [학과명, 총점] 형태로 누적하여 반환한다. 사용자의 가변적인 입력값과 학과의 고정 데이터를 유기적으로 융합하는 핵심적인 역할을 수행한다.

## 5. 조건문과 반복문 활용

[조건문 사용 부분]
어떤 조건을 검사했는가?

print_result() 함수 내부에서 len(results) == 0 조건을 검사하여 분석 결과 리스트가 비어있는 예외 상황인지 판별했다.

또한, 순회 중 각 학과의 final_score 변수 값을 검사하여 점수가 100점 이상(>= 100)인지, 80점 이상 100점 미만(>= 80)인지, 혹은 그 미만인지의 점수 경계 조건을 판별했다.

[반복문 사용 부분]
무엇을 반복 처리했는가?

find_recommendations() 함수 내부에서 2차원 리스트(data)의 행 개수만큼 반복하며 모든 학과의 점수를 자동으로 꺼내어 총점을 연산했다.

print_result() 함수 내부에서 점수 계산이 완료된 결과 리스트를 한 행씩 순서대로 꺼내어 화면에 나열 출력하는 과정을 반복 처리했다.

[조건문과 반복문이 함께 작동한 부분]
print_result() 함수 내의 else 블록 내부에서 두 제어구조가 융합되어 작동한다. for item in results: 반복문이 점수 결과 리스트를 한 줄씩 차례대로 순회하는 과정 속에서, 내부의 if-elif-else 조건문이 현재 순회 중인 학과의 종합 점수를 실시간으로 비교 분석한다. 점수 조건에 맞는 학과에는 행마다 서로 다른 추천 문구([★ 강력 추천!] 또는 [우수 추천])를 동적으로 결합하여 최종 터미널에 출력하는 유기적인 제어 흐름을 완성했다.

## 7. 테스트와 오류 수정


### 테스트 1: 정상 사례

입력: 적성 중요도 5 / 발전가능성 중요도 5 / 흥미 중요도 5 (모든 가치를 최대로 중시하는 경우)

예상 결과: 컴퓨터공학과는 (9*5) + (10*5) + (6*5) = 145점이 나와야 하며, 100점을 넘었으므로 [★ 강력 추천!]이 출력되어야 함.

실제 결과: 예상대로 연산되어 ▶ 컴퓨터공학과 : 145점 [★ 강력 추천!] 문구가 정확하게 화면에 표시됨.


### 테스트 2: 예외 또는 경계 사례

입력: 가수가 없는 텅 빈 학과 데이터셋이 들어오거나 원본 2차원 리스트를 비웠을 때

예상 결과: 반복문 오류로 튕기지 않고, len(results) == 0 안전장치 조건문에 걸려 비정상 데이터 안내 문구가 나와야 함.

실제 결과: 프로그램이 다운되지 않고 분석할 학과 데이터가 존재하지 않습니다.라는 안내가 예외 처리되어 안전하게 출력됨.


### 발견한 오류와 수정한 방법
초기 설계 시 가중치를 입력받는 get_user_input() 함수 내부에 input() 함수만 단독으로 사용했다. 이로 인해 숫자를 입력해도 파이썬이 이를 문자열(String)로 간주하여, find_recommendations() 함수에서 가중합 연산을 시도할 때 문자와 숫자의 곱셈 및 덧셈 에러(TypeError)가 발생하며 프로그램이 강제 종료되는 오류를 발견했다.
이를 수정하기 위해 사용자의 모든 입력 코드 외부를 int(input(...)) 형태로 감싸주어, 입력받은 데이터를 연산이 가능한 정수형 자료형(Integer)으로 명확하게 변환(형변환)해 줌으로써 수리적 가중합 연산 오류를 완벽하게 해결했다.

## 8. Gemini 활용 내용 요약

Gemini에게 도움받은 부분: 자유전공학부의 진로 방향성과 세부 능력을 돋보이게 할 수 있는 융합적 주제(MCDA)를 기획하는 데 도움을 받았다. 또한 학교에서 제공한 미니 프로젝트의 공식 템플릿 골격과 주석 흐름을 해치지 않으면서 필수 감점 방지 루브릭 조건들을 완벽하게 준수하는 직관적인 파이썬 전체 코드의 배치를 설계 받는 데 큰 도움을 얻었다.


Gemini가 제안했지만 내가 수정하거나 사용하지 않은 부분: 가중치를 기반으로 최종 학과 추천 리스트를 점수가 높은 순서대로 내림차순 정렬해 보여주는 복잡한 정렬(sort) 함수 코드를 초기에 예시로 제안받았으나, 고등학교 정보 과목의 기본 교육과정 수준에서 완전히 벗어난 고급 문법이라고 판단했다. 코드 인터뷰 시 완벽한 설명 책임을 완수하기 위해 이를 과감히 제외하고, 순수하게 학교에서 배운 for문과 if 제어구조 정공법만 사용하여 직관적인 순차 출력 코드로 직접 간소화하여 반영했다.


최종 코드에서 내가 직접 판단하거나 수정한 부분: 사용자가 1~5점의 제한 안내 범위를 초과하여 10점이나 100점 등 극단적인 숫자를 입력하더라도 파이썬 내부 연산 메커니즘은 오류 없이 정상 계산된다는 기술적 특징을 직접 분석하여 이해했다. 하지만 가중치 점수의 격차가 너무 벌어지면 학과 고유의 점수 변별력이 깨진다는 점을 깨닫고, 사회과학 조사 기법인 '리커트 5점 척도'의 원리를 적용하여 입력 안내 문구에 (1점 ~ 5점) 기준을 명시하도록 코드의 주석과 안내 인터페이스 문장을 스스로 조율하여 완성도를 높였다.

## 9. 한계와 개선 방안

현재 프로그램의 한계: 현재는 사용자가 1~5 범위를 완전히 벗어난 에러성 데이터(예: 문자열 "오점" 또는 숫자가 아닌 데이터)를 기입했을 때, 코드 내부에서 이를 걸러내고 다시 입력받게 만드는 '입력 유효성 검사 루프'가 누락되어 있어 ValueError로 멈출 가능성이 존재한다.


다시 개선한다면 추가하고 싶은 기능: 다시 개선한다면 추가하고 싶은 기능: 사용자가 1~5점 이외의 문자나 잘못된 범위를 입력했을 때, while 반복문과 조건문을 결합하여 "잘못된 입력입니다. 1~5 사이의 숫자를 다시 입력하세요"라고 경고를 보내고 올바른 값을 넣을 때까지 계속 입력을 다시 받는 '무한 루프 방어 코드(Input Validation)' 기능을 추가하여 프로그램의 안정성을 대폭 업그레이드하고 싶다.
